# 库存管理系统 - 前端使用说明

## 📋 系统概述

这是一个现代化的库存管理系统前端界面，采用 Apple 风格设计，集成了完整的 API 对接功能。

## 🎨 设计特点

- **Apple 风格设计**: 采用圆角、毛玻璃效果、优雅的动画过渡
- **响应式布局**: 完美支持桌面端和移动端
- **TailwindCSS**: 使用 TailwindCSS 进行样式管理
- **ECharts 图表**: 数据可视化展示
- **Font Awesome 图标**: 丰富的图标库

## 📁 页面结构

系统包含以下 9 个主要页面：

### 1. 仪表盘 (Dashboard)
- **路径**: `#dashboard-page`
- **功能**: 
  - 显示库存统计数据（总物品、低库存、库存价值、周转率）
  - 库存趋势图表
  - 类别分布饼图
  - 最近活动记录
  - 低库存物品列表

### 2. 库存物品 (Items)
- **路径**: `#items-page`
- **功能**:
  - 物品列表展示（带分页）
  - 搜索和筛选功能
  - 添加/编辑/删除物品
  - 库存预警提示

### 3. 入库管理 (Inbound)
- **路径**: `#inbound-page`
- **功能**:
  - 入库统计（今日/本周/本月）
  - 入库记录列表
  - 新增入库操作

### 4. 出库管理 (Outbound)
- **路径**: `#outbound-page`
- **功能**:
  - 出库统计（今日/本周/本月）
  - 出库记录列表
  - 新增出库操作

### 5. 库存调拨 (Transfer)
- **路径**: `#transfer-page`
- **功能**:
  - 仓库间物品调拨
  - 调拨记录查询

### 6. 仓库管理 (Warehouse)
- **路径**: `#warehouse-page`
- **功能**:
  - 仓库信息展示
  - 仓库使用率统计
  - 仓库位置管理

### 7. 供应商管理 (Supplier)
- **路径**: `#supplier-page`
- **功能**:
  - 供应商列表
  - 供应商信息管理
  - 联系方式维护

### 8. 报表分析 (Reports)
- **路径**: `#reports-page`
- **功能**:
  - 库存周转分析图表
  - 出入库对比图表
  - 数据导出（Excel/PDF）

### 9. 系统设置 (Settings)
- **路径**: `#settings-page`
- **功能**:
  - 基本信息设置
  - 库存预警配置
  - 账户信息管理
  - 系统信息查看

## 🔌 API 集成

### API 配置

系统内置 API 服务层，配置位于 `API_CONFIG` 对象：

```javascript
const API_CONFIG = {
  baseURL: 'http://localhost:8000/api',
  token: localStorage.getItem('auth_token') || ''
};
```

### API 服务方法

#### 仪表盘 API
```javascript
API.dashboard.getStats()                    // 获取统计数据
API.dashboard.getTrend('monthly')           // 获取趋势数据
API.dashboard.getCategoryDistribution()     // 获取类别分布
API.dashboard.getRecentActivity(10)         // 获取最近活动
API.dashboard.getLowStock()                 // 获取低库存物品
```

#### 物品管理 API
```javascript
API.items.list({ page: 1, search: '鼠标' }) // 获取物品列表
API.items.get(1)                             // 获取物品详情
API.items.create(data)                       // 创建物品
API.items.update(1, data)                    // 更新物品
API.items.delete(1)                          // 删除物品
```

#### 出入库 API
```javascript
API.inventory.inbound(data)                  // 入库操作
API.inventory.outbound(data)                 // 出库操作
API.inventory.transfer(data)                 // 调拨操作
API.inventory.history({ item_id: 1 })       // 获取历史记录
```

### 演示模式

系统内置演示模式，当 API 请求失败时会自动返回模拟数据，方便前端开发和测试。

```bash

index.html
```

### 2. 本地服务器
```bash

# 使用 Node.js
npx http-server -p 8080
```

### 3. 集成后端
确保后端 API 服务运行在 `http://localhost:8000`，前端会自动连接。

编辑 `index.html` 中的 API 配置：
```javascript
const API_CONFIG = {
  baseURL: 'https://your-api-domain.com/api',
  token: localStorage.getItem('auth_token') || ''
};
```

### 修改主题颜色
在 TailwindCSS 配置中修改：
```javascript
tailwind.config = {
  theme: {
    extend: {
      colors: {
        primary: '#007AFF',    // 主色调
        secondary: '#5AC8FA',  // 次要色

        warning: '#FF9500',    // 警告色
        danger: '#FF3B30',     // 危险色
      }
    }
  }
};
```

## 📊 数据流程

### 页面加载流程
1. 用户点击导航菜单
2. `PageManager.showPage()` 隐藏其他页面，显示目标页面
3. 调用对应的数据加载函数（如 `loadItemsPage()`）
4. 发起 API 请求获取数据
5. 渲染数据到页面

### API 请求流程
1. 调用 `API.request()` 方法
2. 自动添加认证 Token
3. 发送 HTTP 请求
4. 处理响应或错误
5. 失败时返回模拟数据（演示模式）

## 🛠️ 开发建议

### 添加新页面
1. 在 `<main>` 中添加新的页面 HTML
2. 在 `PageManager.pages` 中注册页面
3. 创建数据加载函数
4. 在侧边栏添加导航链接

### 添加新 API
1. 在 `API` 对象中添加新的方法组
2. 实现具体的 API 调用
3. 在 `getMockData()` 中添加模拟数据

### 调试技巧
- 打开浏览器控制台查看 API 请求
- 检查 `console.log('库存管理系统已就绪 - API集成完成')` 确认初始化
- 使用浏览器开发者工具查看网络请求

## 📝 注意事项

1. **跨域问题**: 如果 API 和前端不在同一域名，需要配置 CORS
2. **Token 管理**: Token 存储在 localStorage，需要实现登录功能来设置
3. **图片上传**: 目前使用占位图片，需要实现真实的图片上传功能
4. **分页功能**: 已预留分页容器，需要实现分页逻辑
5. **表单验证**: 添加物品表单需要添加客户端验证

## 🔐 安全建议

- 不要在前端代码中硬编码敏感信息
- 使用 HTTPS 传输数据
- 实现 Token 过期刷新机制
- 对用户输入进行验证和清理
- 实现适当的权限控制
